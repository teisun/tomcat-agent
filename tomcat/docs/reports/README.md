# reports/ — 调研归档

本目录存放**一次性或阶段性的调研、对比、根因分析、方案探索**等文档。内容多为 Agent 在实现某任务前对外部项目、竞品能力或历史问题的背景调查，**不是**项目的规范、架构说明或开发手册。

---

## 开发者需要关心吗？

**通常不需要。**

日常开发、修 bug、写测试、对齐 OpenSpec 时，请以以下目录为准：

| 目录 | 用途 |
|------|------|
| `docs/openspec/` | 规范、架构、变更与验收标准 |
| `docs/architecture/` | 模块级技术说明 |
| `docs/agents/` | 任务看板与执行剧本 |
| `src/` | 源码与内联文档 |

本目录下的报告**不保证与当前代码同步**，也**不具约束力**。若某结论已落地，以 OpenSpec / 源码 / 对应 architecture 文档为准；若未落地，可当作背景参考，不必逐篇阅读。

---

## 文档大致类型

目录内文件命名不统一，但大体可归为：

- **竞品 / 跨仓对比** — 如 Agent 工具实现、Plan Mode、多 Session、插件体系等横向调研
- **差距与兼容性分析** — 如 extension API、pi-mono、Wasm 迁移等
- **根因与专项分析** — 如 tool message 链 violation、集成测试 hang、VM shutdown 等
- **方案探索与 playbook** — 如 context 管理重构提案、Plan Mode 执行手册等
- **验收 / 复盘报告** — 如 E2E recap、社区 E2E 验收等

---

## 何时可能需要打开

- 追溯某个设计决策的**原始调研依据**
- 新写类似主题的调研，避免重复劳动
- 任务卡或 PR 描述中引用了某份 report 作为上下文

除此以外，**可以安全忽略本目录**。
