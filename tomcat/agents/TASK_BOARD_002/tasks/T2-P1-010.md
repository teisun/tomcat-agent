**目标**：按架构文档落地 transcript-first reasoning continuity：同 provider 高保真 replay，跨 provider 可 downgrade，会话不断。

**范围**：

- 只做 continuity 的 capture / persist / replay / downgrade / scrubber。
- 不做 CLI / TUI thinking 展示改造。
- 不把 `previous_response_id` 当主路线；OpenAI 主线仍是 `store=false` + 显式 replay。

**子项**（与架构文档 `§4.1` 选型决策表、`§4.2` 实施点一致）：

- [x] **schema-and-policy**（`R1-R3` / `P1`）：落 `TranscriptTurnRecord`、`ReasoningContinuation`、`ProviderCompatProfile`、`ReplayPolicy`；替换粗粒度 resend 规则
- [x] **openai-responses**（`R4` / `P2`）：实现 `store=false` + `include=["reasoning.encrypted_content"]` 的 capture / replay 主线；`previous_response_id` 仅作可选且互斥的 `store=true` 分支
- [x] **deepseek-replay-profile**（`R5` / `P3`）：区分 `deepseek-reasoner` 与 `deepseek-chat` / `deepseek-v4-*` thinking mode；兼容 `reasoning_content` 默认回放，tool-call turn 继续记录更强 replay 约束
- [x] **downgrade-and-legacy**（`R6` / `P4`）：实现 cross-provider `keep / convert / strip`，优先 `thinking_text`，兼容无 continuity 字段的老 transcript
- [x] **scrubber-and-strip**（`R7` / `P5`）：补 transport strip 与 think scrubber，避免 `<think>` 等内容泄漏到正文
- [x] **tests-and-doc-sync**（`P6`）：补齐测试矩阵并同步相邻文档边界（已完成 `deepseek-v4-pro` same-model + `v4-pro -> v4-flash` switch-model real-LLM 验收）

**依赖**：

- 现有 `OpenAiResponsesProvider`、`openai.rs`、JSONL transcript、`StreamEvent::Thinking` 为前置
- 无新增开放任务硬阻塞

**协作接口**：

- `src/core/llm/types.rs`
- `src/core/session/transcript.rs`
- `src/core/session/manager/context.rs`
- `src/core/llm/replay_policy.rs`（建议新增）
- `src/core/llm/openai_responses/mod.rs`
- `src/core/llm/openai_responses/stream.rs`
- `src/core/llm/openai_responses/payload.rs`
- `src/core/llm/openai.rs`

**验收标准**：

- 通过架构文档 `§11` 的核心锚点：`transcript_roundtrip_preserves_reasoning_continuation`、`openai_responses_roundtrip_replays_reasoning_items`、`deepseek_tool_turn_replays_reasoning_content`、`deepseek_non_tool_turn_roundtrip_replays_reasoning_content`、`cross_provider_downgrade_keeps_semantic_history`、`streaming_think_scrubber_hides_split_tags`
- OpenAI 主线保持 `store=false` + 显式 replay；`previous_response_id` 仅在显式配置下切到互斥分支
- DeepSeek continuity snapshot 无论 tool turn 与否都可稳定持久化；同 family 兼容 `reasoning_content` 默认回放；tool-call turn 继续以 metadata 表达更强 replay 约束；`deepseek-v4-pro` / `deepseek-v4-flash` 进入同一 DeepSeek V4 continuity family
- 默认行为不破坏现网；老 transcript 无 continuity 字段仍可继续会话
- 本轮只补 `SessionEntry.model_override` + `model_change` 的最小切换链路，供 integration / 审计使用；完整多 LLM 配置与会话内产品化切换不在本卡范围
- `cargo fmt --check`、`cargo clippy --all-targets -- -D warnings`、相关 `cargo test` 全绿
