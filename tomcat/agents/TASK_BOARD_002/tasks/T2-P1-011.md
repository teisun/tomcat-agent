**目标**：为 Tomcat 补齐 **current-tail aggregate overflow** 的正式预防链路：在 `reasoning_loop` 每次工具循环结束、发下一次 `llm.chat_stream(...)` 前做 aggregate precheck；若已超预算，则先 apply 已完成历史摘要，再把 apply 后历史整体压一遍，最后才对 current tail 的 `COMPACTABLE_TOOLS`（首版 `read` / `search_files` / `bash` / `task_output`）做“旧的一半 -> 再旧的一半”波次 placeholder；若仍不够，则直接复用既有 `branch_summary + apply_boundary` 把整份 working messages 折成单条摘要，并拼接 keepalive snapshot。同批收口 `keep_recent_turns` 接线与 `compaction_turns` 删除。

**子项**（与计划 frontmatter `todos`、架构文档 §4.2 对齐；认领时可按 PR-CTA / CTB / CTC / CTD 拆分）：

- [x] **wire-keep-recent**：`truncation.rs` 的 L1 保护区改读 `config.keep_recent_turns`（目标默认值 `5`），删除 `M_PROTECTED_TURNS` 常量依赖；同步 `context-management.md` / `core/README.md`
- [x] **remove-compaction-turns**：删除 `context.compaction_turns`（`ContextConfig`、config allowlist、catalog 文案、单测）
- [x] **mid-turn-precheck**：在 `reasoning_loop` 每次 tool round 结束、发 LLM 前执行 aggregate precheck；产出 `fits` / `reduce_current_tail` / `collapse_to_branch_summary`
- [x] **aggregate-reduction**：新增 `current_tail_guard` 路径；首版固定 `COMPACTABLE_TOOLS = {read, search_files, bash, task_output}`；先 apply 已完成历史摘要，再对 apply 后历史整体再压一遍，最后对 tail 做“旧的一半 -> 再旧的一半”波次 placeholder
- [x] **rewrite-and-helper-reuse**：复用/抽出 `truncation.rs` 现有“大 result 落盘 + placeholder” helper，使 mid-turn reduction 可传可控阈值；由 `session/transcript` 模块提供“按 `msg_id` 重写 message”的 helper 做 best-effort 回写；动到 apply 后历史消息时正确 `invalidate_api_usage()`
- [x] **collapse-to-branch-summary**：当 B 仍不够时，直接复用既有 `branch_summary + apply_boundary` 把整份 working messages 压成单条摘要；不再设计 `current_tail.split_turn` 自定义事件
- [x] **state-keepalive**：collapse/reduction 路径保活执行态：`plan mode`、`active_plan_path`、当前 step、关键 pending work、`latest_plan_event`
- [x] **tests-and-doc-sync**：补单测/集成测/文档同步：many medium reads 累积 precheck、apply 后历史再压、mixed tail 只改 `COMPACTABLE_TOOLS`、single branch_summary collapse、长 plan 连续性、仅压老历史仍会爆窗的负向用例

**协作接口**：

- `reasoning_loop.rs`：mid-turn precheck 入口与 route
- `tool_dispatcher.rs`：tool result eager append、`on_message_appended(...)` 计费基线
- `accessors.rs`：`messages[start_idx..]` 的 current tail 视角
- `current_tail_guard.rs`【新】：估算可减空间、先 apply 历史、再压历史、再做 tail reduction，并决定何时进入单条 branch_summary collapse
- `apply.rs`：复用既有 `apply_boundary` 路径吃掉已完成历史摘要，并用于 C 的单条 branch_summary apply
- `session/transcript.rs`：提供按 `msg_id` 重写 message 的 helper；reduction 通过该 helper best-effort 回写；C 继续复用既有 `BranchSummaryEntry` 语义
- `context.rs` / `types.rs`：history apply / collapse 后的消息折叠、`post_usage_appended_chars` / `invalidate_api_usage()`
- `preheat.rs`：复用 structured summary 家族生成单条 branch_summary 正文
- `plan_runtime/mod.rs`：keepalive snapshot 来源
- `truncation.rs` / `ContextConfig` / allowlist / catalog：配置清理，以及复用/抽出 L0/L1 helper

**验收标准**：

- `reasoning_loop` 在**每次** tool round 结束后、下一次 `llm.chat_stream(...)` 前执行 current-tail precheck；many-medium-reads 场景在发请求前进入 reduction/collapse，而不是先撞 API
- B 会先吃掉现成历史 `branch_summary` 收益，再对 apply 后历史整体再压一遍，最后才对 tail 里的 `COMPACTABLE_TOOLS` 做“旧的一半 -> 再旧的一半”波次 placeholder；非白名单结果不被误改
- reduction 在当前轮内**总是**先更新局部 `messages` 与计数；随后通过 transcript helper best-effort 回写；若动到 apply 后历史消息则正确 `invalidate_api_usage()`
- C 成功后 working messages 被折成单条 `branch_summary + keepalive snapshot`；`PlanRuntime` 保持正确执行态，`current step` / `pending work` 可继续推进；`start_idx` / `context_tail_start` 与 token 基线同步更新
- `keep_recent_turns` 真正生效；`compaction_turns` 从类型、allowlist、catalog、单测、README 中全部移除
- 交付前通过 `cargo fmt --check`、`cargo clippy --all-targets -- -D warnings`、相关 `cargo test`；并按 [INTEGRATION_MERGE_AND_ACCEPTANCE.md](../../INTEGRATION_MERGE_AND_ACCEPTANCE.md) 完成分类集成与全量验收

**已落地备忘（实施本卡前勿删，便于对照）**：

- 当前 `ContextState::estimated_token_count()` 已使用 `last_api_usage + post_usage_appended_chars / 4`；所以 B/C 若改写 tail，**不能**只改消息文本，必须同步修正计数
- 当前 `tool_dispatcher.rs` 会在工具执行后立即 `push_message(...)` 并 `ctx_state.on_message_appended(...)`；这意味着阶段二即使写盘失败，也必须先把局部 `messages` 与计数减下来，避免当前轮继续超重；同时仍应通过 transcript helper best-effort 回写，减小 reload 漂移
- 当前 `truncation.rs` 仍有 `M_PROTECTED_TURNS = 5`，`ContextConfig` 仍带 `compaction_turns`，说明配置层与 runtime 仍未完全收口
- 当前仓库的 `BranchSummaryEntry` / `apply_boundary()` 已是成熟路径；本卡的 C 路径**直接复用它**做单条 branch_summary collapse，不再新增 `split-turn` 自定义事件
