# Proposal: 一期 MVP 核心引擎与插件系统落地
## 提案背景
项目处于从零启动阶段，核心目标是打造一款基于Rust+WasmEdge+QuickJS的pi-mono生态兼容AI Agent运行时，解决pi-mono原生实现的安全风险问题，同时100%兼容社区插件生态。一期MVP需要优先落地核心宿主引擎、WasmEdge沙箱插件系统、pi-mono兼容的4原语、事件系统、宿主API，跑通「插件加载→执行→宿主API调用→对话交互」的最小闭环，验证架构可行性与生态兼容性。所有设计严格对齐pi-agent-rust与pi-mono规范，遵循安全隔离优先的核心原则。

## 核心目标
1.  完成Rust宿主核心架构落地，实现基础设施层、事件总线、配置管理、日志审计系统，层间边界清晰，无循环依赖
2.  完成全局WasmEdge运行时引擎初始化与生命周期管理，实现独立Wasm实例沙箱隔离机制，内置官方QuickJS运行时
3.  完成pi-mono 100%兼容的4原语宿主API落地，实现read/write/edit/bash核心能力，配套权限管控、用户确认、审计日志全流程安全机制
4.  完成Node.js核心兼容层落地，覆盖pi插件高频使用的核心模块与全局对象，保证社区插件兼容性
5.  完成宿主核心API体系落地，包含LLM统一接入、工具注册中心、事件系统、会话管理、插件生命周期管理，完全对齐pi-mono ExtensionAPI规范
6.  完成插件全生命周期管理系统落地，实现插件的加载、初始化、启用、禁用、卸载，独立沙箱隔离，错误完全隔离
7.  完成极简CLI工具落地，实现引擎初始化、配置管理、会话管理、对话交互、插件管理核心命令
8.  完成基础安全体系落地，实现插件级细粒度权限管控、沙箱隔离、全链路审计日志（敏感数据加密 TODO 后续考虑）
9.  验证pi-mono社区插件兼容性，标准pi插件零修改即可正常加载运行
10. 所有功能严格遵循安全隔离优先原则，无越权操作、无静默执行、无宿主系统污染

## 不做的范围
1.  插件自举全闭环、AI自主生成插件（二期实现）
2.  多 Agent 完整体系、多 channel 实现、自定义 Agent 能力（三期实现）
3.  长程记忆系统（四期实现）
4.  Skills技能工作流系统（五期实现）
5.  Web/Android前端界面（六期实现）
6.  容器化运行环境、多平台交叉编译（三期实现）
7.  插件市场、多模态能力、团队协作（长期规划）

## 风险与应对方案
| 风险描述 | 影响等级 | 应对方案 |
|----------|----------|----------|
| WasmEdge与QuickJS集成复杂，pi-mono API兼容难度高 | 高 | 优先使用WasmEdge官方原生JS运行时扩展，开箱即用，无需手动处理引擎打包；先对齐pi-mono核心高频API，再逐步覆盖边缘API，分阶段验证兼容性；以官方pi-mono插件作为兼容性测试用例 |
| 4原语安全管控设计复杂，平衡便利性与安全性难度高 | 中 | 参考pi-agent-rust的安全设计，默认采用最严格的权限策略，路径白名单+命令分级管控，所有高危操作默认需要用户确认，提供配置项让用户可按需放宽策略，兼顾不同用户需求 |
| Node.js兼容层覆盖不全，导致社区插件无法正常运行 | 高 | 优先覆盖pi插件高频使用的核心模块（fs/path/process/console/http），基于WasmEdge官方原生实现，保证API行为与Node.js完全对齐；建立兼容性测试集，覆盖主流社区插件的依赖场景 |
| 事件系统与pi-mono行为不一致，导致插件事件监听异常 | 中 | 完全对齐pi-mono的事件枚举、回调参数、执行顺序，先完成事件总线核心设计，再做上层业务事件，每一个事件都做兼容性测试，保证行为100%一致 |
| 跨平台WasmEdge环境适配复杂，不同平台兼容性问题 | 中 | 采用WasmEdge官方跨平台预编译库，封装统一的平台适配层，抹平平台差异；提前搭建三大桌面平台的CI测试环境，每一次提交都做跨平台编译验证 |

## 验收标准
1.  宿主引擎可正常编译、稳定启动、关闭，无崩溃、无致命错误日志，跨Windows/macOS/Linux三大平台正常运行
2.  `pi-awsm init`命令可正常完成初始化配置，LLM API密钥可配置并生效（加密存储 TODO 后续考虑）
3.  4原语核心能力完整可用，完全对齐pi-mono API规范，权限管控、用户确认、审计日志机制正常生效
4.  WasmEdge沙箱插件系统正常运行，可正常加载、执行、卸载JS/TS插件，每个插件独立实例隔离，错误完全隔离不影响宿主
5.  pi-mono标准社区插件零修改即可正常加载运行，核心API调用正常，Node.js兼容层无兼容性问题
6.  事件系统正常运行，插件可正常监听内置事件、发布自定义事件，回调执行正常，错误不影响主流程
7.  工具注册API正常可用，插件可注册自定义工具，Agent对话中可正常调用，权限校验、审计日志正常
8.  LLM统一接入正常，可正常配置多模型，流式/非流式调用正常，Token统计准确
9.  CLI工具核心命令全部可用，对话模式交互流畅，会话管理正常：元数据 store（sessions.json）与 pi 系 JSONL transcript 两层落盘，列表与路由由 sessions.json 提供，会话内容仅存 JSONL；
10. 核心模块单元测试覆盖率≥80%，所有测试用例全部通过，代码通过clippy全量规则校验，无警告
11. 所有功能严格遵循安全设计原则，无越权操作、无静默高危操作、无宿主系统污染，审计日志完整可追溯
12. 配套的设计文档、代码注释、用户使用说明完整清晰，可维护性强
