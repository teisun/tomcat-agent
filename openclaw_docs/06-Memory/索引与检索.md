# Memory 索引与检索

## 零、先用大白话

向量检索像 **按意思找相似便利贴**。  
关键词检索像 **按字搜标签**。  
**hybrid** 把两堆结果 **混一起重新排队**，取最可能有用的一小段给模型。

**这一节你会学到**：chunk 是啥；向量与 FTS 各管啥。

---

**设计思想**：Memory 使用 sqlite-vec 做向量索引，SQLite FTS5 做 BM25 关键词检索，两者结果经 `mergeHybridResults` 合并为 hybrid 排序结果。

## 一、Schema 与 Chunk

- **Schema**：**`ensureMemoryIndexSchema`** 定义在 **`src/memory-host-sdk/host/memory-schema.ts`**（宿主侧）并由 **`extensions/memory-core/`** 内 manager 复用/衔接；创建表结构的具体调用见 **`extensions/memory-core/src/memory/manager-sync-ops.ts`** 等。
- **chunk**：Markdown 按段落或块切分（实现位于 **`extensions/memory-core/src/memory/`** 各模块）；`MemoryChunk` 含 path、行范围、content、embedding 等字段。

---

## 二、检索

- **searchVector**：**`extensions/memory-core/src/memory/manager-search.ts`**，sqlite-vec 向量相似度搜索。
- **searchKeyword**：FTS5 全文搜索，`buildFtsQuery` 构建查询（同在 memory-core 包内检索模块）。
- **mergeHybridResults**：**`extensions/memory-core/src/memory/hybrid.ts`**，按 vectorWeight、textWeight 合并并排序。

---

## 三、数据源

- **memory**：MEMORY.md、workspace/memory/*.md
- **sessions**：stateDir/agents/<agentId>/sessions/*.jsonl 转录

## 常见误会

- **误会**：向量搜一定最准。**正解**：专有名词、人名、命令行片段有时 **BM25 更稳**；所以才有 hybrid。  
- **误会**：命中越多上下文越长越好。**正解**：只取 **topK 片段**；太多会挤爆模型窗口。  
- **误会**：索引文件随便拷到另一台就能用。**正解**：路径、agentId、embedding 模型要 **一致**；否则要重建。
