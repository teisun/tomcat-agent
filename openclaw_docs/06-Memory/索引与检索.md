# Memory 索引与检索

**设计思想**：Memory 使用 sqlite-vec 做向量索引，SQLite FTS5 做 BM25 关键词检索，两者结果经 `mergeHybridResults` 合并为 hybrid 排序结果。

---

## 一、Schema 与 Chunk

- **memory-schema.ts**：`ensureMemoryIndexSchema` 创建表结构。
- **chunk**：Markdown 按段落或块切分，`chunkMarkdown`、`listMemoryFiles`。
- **MemoryChunk**：path、startLine、endLine、content、embedding 等。

---

## 二、检索

- **searchVector**：`openclaw/src/memory/manager-search.ts`，sqlite-vec 向量相似度搜索。
- **searchKeyword**：FTS5 全文搜索，`buildFtsQuery` 构建查询。
- **mergeHybridResults**：`openclaw/src/memory/hybrid.ts`，按 vectorWeight、textWeight 合并并排序。

---

## 三、数据源

- **memory**：MEMORY.md、workspace/memory/*.md
- **sessions**：stateDir/agents/<agentId>/sessions/*.jsonl 转录
