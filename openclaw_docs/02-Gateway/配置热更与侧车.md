# Gateway 配置热更与侧车

**设计思想**：Gateway 通过 `startGatewayConfigReloader` 监听配置文件变化，按 `GatewayReloadPlan` 决定 hot reload 或 full restart。Browser、Canvas-host、Tailscale 等作为侧车由 `startGatewaySidecars` 启动，随 Gateway 生命周期管理。

---

## 一、配置热更

**入口**：`openclaw/src/gateway/config-reload.ts` 的 `startGatewayConfigReloader`

**流程**：

1. 使用 chokidar 监听配置文件路径。
2. 文件变化时 debounce（默认 300ms）。
3. 调用 `computeGatewayReloadPlan` 根据变化路径决定 reload 类型：
   - **hot**：仅重载部分组件（hooks、cron、heartbeat、browser、channels 等）。
   - **restart**：重启整个 Gateway。
   - **none**：忽略（如 gateway.remote、wizard、logging）。

**Reload 规则**：`BASE_RELOAD_RULES` 定义前缀与动作，例如：

- `hooks` → reload-hooks
- `cron` → restart-cron
- `browser` → restart-browser-control
- `plugins` → restart Gateway

---

## 二、侧车启动顺序

**入口**：`openclaw/src/gateway/server-startup.ts` 的 `startGatewaySidecars`

**侧车列表**（按配置启用）：

1. **Browser**：`openclaw/src/gateway/server-browser.ts` 的 `startBrowserControlServerIfEnabled`，启动 Browser 控制服务。
2. **Canvas-host**：`openclaw/src/canvas-host/server.js`，A2UI 宿主。
3. **Tailscale**：`openclaw/src/gateway/server-tailscale.ts` 的 `startGatewayTailscaleExposure`，Serve/Funnel 暴露。

**启动依赖**：部分侧车依赖 Gateway port 已绑定，通过 `CLAWDBOT_GATEWAY_PORT` 等环境变量传递。

---

## 三、Browser 侧车

- **server-browser.ts**：按 `gateway.browser` 或 `tools.browser` 配置决定是否启动。
- **browser/server.ts**：Browser 控制 HTTP 服务，提供 snapshot、act 等端点。
- **与 browser-tool 协作**：Agent 调用 browser 工具时，通过 Gateway 或直接 HTTP 访问 Browser 服务。

---

## 四、Canvas-host 侧车

- **canvas-host/server**：提供 A2UI 渲染与 push/reset 等能力。
- **Gateway 协议**：canvas 相关 method 通过 Gateway 转发到 canvas-host。

---

## 五、Tailscale 侧车

- **server-tailscale.ts**：根据 `gateway.tailscale.mode`（off/serve/funnel）配置 Tailscale Serve 或 Funnel。
- **bind 约束**：启用 Tailscale 时，Gateway 必须 bind loopback，由 Tailscale 做反向代理。
