# Gateway 配置热更与侧车

## 零、先用大白话

热更新像 **飞机还在飞，就换地图**。  
你改了 `openclaw.json`，Gateway 尽量 **不把整台机器关机**，只换该换的那几块。  
**侧车**像 **跟着飞的加油机**：浏览器控制、Tailscale 暴露、插件服务……主飞机在，它们才跟着有意义。

**这一节你会学到**：哪些改动只要「热一下」；哪些必须整进程重启；侧车从哪个文件启动。

---

**设计思想**：Gateway 通过 `startGatewayConfigReloader` 监听配置文件变化，按 `GatewayReloadPlan` 决定 hot reload 或 full restart。侧车与 channel 启动等多在 `startGatewaySidecars` / post-attach 流程里编排（见下文路径）。

## 一、配置热更

**入口**：`src/gateway/config-reload.ts` 的 `startGatewayConfigReloader`

**流程**：

1. 使用 chokidar 监听配置文件路径。
2. 文件变化时 debounce（默认 300ms）。
3. 调用 `computeGatewayReloadPlan` 根据变化路径决定 reload 类型：
   - **hot**：仅重载部分组件（hooks、cron、heartbeat、browser、channels 等）。
   - **restart**：重启整个 Gateway。
   - **none**：忽略（如 gateway.remote、wizard、logging）。

**Reload 规则**：`BASE_RELOAD_RULES` 定义前缀与动作，例如：

- `hooks` → reload-hooks
- `cron` → restart-cron
- `browser` → restart-browser-control
- `plugins` → restart Gateway

---

## 二、侧车与「启动后半场」

**侧车 / channels 等编排入口**：`src/gateway/server-startup-post-attach.ts` 里的 `startGatewaySidecars`（名字以仓库为准；不要把旧文档里的 `server-startup.ts` 背死）。

**常见侧车或附属能力**（是否启用看配置）：

1. **Browser 控制**：`src/gateway/server-browser.ts` 的 `startBrowserControlServerIfEnabled`。  
2. **Canvas / A2UI 宿主**：`src/canvas-host/server.ts`（TypeScript 源码；与 Gateway HTTP 同端口上的路径前缀见上游 `docs/concepts/architecture.md`）。  
3. **Tailscale 暴露**：`src/gateway/server-tailscale.ts` 的 `startGatewayTailscaleExposure`（Serve/Funnel）。

**端口**：Gateway 绑端口后会把 `OPENCLAW_GATEWAY_PORT` 设成字符串，方便浏览器派生端口等（见 `src/gateway/server.impl.ts`）。解析端口还可看 `src/config/paths.ts` 里的 `resolveGatewayPort`。

---

## 三、Browser 与「浏览器控制」

名字和文件会在版本里微调。**实用找法**：在 `openclaw` 仓库根对 **`browser`**、`BrowserControl`、`cdp` 做搜索；配置热更规则里常出现 **`browser` → restart-browser-control** 一类条目（见 `src/gateway/config-reload.ts` 的 `BASE_RELOAD_RULES`）。  
Agent 侧工具通过 Gateway 或本地 HTTP 访问浏览器控制端点（以当前 `src/agents/tools` 与官方文档为准）。

---

## 四、Canvas-host 侧车

- **`src/canvas-host/server.ts`**：A2UI / canvas 宿主逻辑入口（旁边还有 `a2ui.ts` 等）。  
- **Gateway 协议**：canvas 相关 method 经 Gateway 转发到宿主（细节搜 `canvas` 于 `src/gateway`）。

---

## 五、Tailscale 侧车

- **`src/gateway/server-tailscale.ts`**：按 `gateway.tailscale.mode`（off/serve/funnel）走 Serve 或 Funnel。  
- **bind 约束**：启用 Tailscale 时，Gateway 通常 **只绑本机回环**，由 Tailscale 做反向代理（见上游网络模型文档）。

---

## 常见误会

- **误会**：改任何配置都是 hot reload。**正解**：`GatewayReloadPlan` 里可能写 **restart**；改到 `plugins` 一类甚至会整网关重启。  
- **误会**：侧车挂了主 Gateway 一定挂。**正解**：不少侧车 **best-effort**；看日志里 warn，别想当然。  
- **误会**：Tailscale 打开就能从公网随便进。**正解**：还有 **鉴权、配对、防火墙**；读官方安全章节。
