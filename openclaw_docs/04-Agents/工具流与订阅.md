# Agents 工具流与订阅

## 零、先用大白话

工具流像 **助理边想边查资料**。  
模型说：我要用某个工具。  
工具跑完，结果要 **回到同一条故事线**，UI 才不会断章。

**这一节你会学到**：回调名字；工具文件分散在哪；memory 相关为啥不叫单一 `memory-tool.ts`。

---

**设计思想**：Pi 在运行中会调用工具，工具结果通过 `onToolResult` 等回调返回。工具清单与策略见 **`src/agents/tool-catalog.ts`** 等；具体实现分散在 **`src/agents/tools/*.ts`**（如 `gateway-tool.ts`、`sessions-*-tool.ts`、`browser` 相关工具等）；**`memory_get` / `memory_search`** 等由 Pi 工具策略与嵌入式订阅协同，而非单一 `memory-tool.ts` 文件名。

---

---

## 一、回调

- **onToolResult**：工具执行完成，结果回传 Pi。
- **onPartialReply**：流式文本片段。
- **onBlockReply**：块级回复（如代码块）。
- **onReasoningStream**：思考过程流（若启用）。

---

## 二、工具实现

| 工具 | 文件 | 职责 |
|------|------|------|
| gateway-tool | `src/agents/tools/gateway-tool.ts` | 调用 Gateway methods |
| sessions_list | `src/agents/tools/sessions-list-tool.ts` | 列出会话 |
| sessions_history | `src/agents/tools/sessions-history-tool.ts` | 获取历史 |
| sessions_send | `src/agents/tools/sessions-send-tool.ts` | 跨会话发送 |
| memory_get / memory_search | `src/agents/tool-catalog.ts` + Pi 嵌入式订阅 | Memory 读取/检索（名称以 catalog 为准） |
| browser / image 等 | `src/agents/tools/` 下对应文件 | 浏览器、生图等（以目录为准） |

---

## 三、与 Gateway 协作

- gateway-tool 通过 GatewayClient 或 callGateway 调用 Gateway methods。  
- 部分工具（如 send）需 Gateway 的 send method 完成出站。

---

## 常见误会

- **误会**：工具返回啥就直接给用户看。**正解**：还要经 **策略、脱敏、渠道格式** 等几层（出错看日志）。  
- **误会**：`gateway-tool` 可以随便调任意 method。**正解**：仍受 **鉴权、tool-policy、审批** 约束。  
- **误会**：关掉所有 tools 就最省 token。**正解**：有些 **系统能力** 仍会通过别路径注入；要读配置项。
