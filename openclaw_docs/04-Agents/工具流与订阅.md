# Agents 工具流与订阅

**设计思想**：Pi 在运行中会调用工具，工具结果通过 `onToolResult` 等回调返回。OpenClaw 实现的工具（gateway-tool、sessions-*、memory-tool、browser-tool）通过 Pi 的 tool 机制注册，执行时可能调用 Gateway、Memory、Browser 等服务。

---

## 一、回调

- **onToolResult**：工具执行完成，结果回传 Pi。
- **onPartialReply**：流式文本片段。
- **onBlockReply**：块级回复（如代码块）。
- **onReasoningStream**：思考过程流（若启用）。

---

## 二、工具实现

| 工具 | 文件 | 职责 |
|------|------|------|
| gateway-tool | agents/tools/gateway-tool.ts | 调用 Gateway methods |
| sessions_list | agents/tools/sessions-list-tool.ts | 列出会话 |
| sessions_history | agents/tools/sessions-history-tool.ts | 获取历史 |
| sessions_send | agents/tools/sessions-send-tool.ts | 跨会话发送 |
| memory-tool | agents/tools/memory-tool.ts | Memory 检索 |
| browser-tool | agents/tools/browser-tool.ts | Browser 控制 |

---

## 三、与 Gateway 协作

- gateway-tool 通过 GatewayClient 或 callGateway 调用 Gateway methods。
- 部分工具（如 send）需 Gateway 的 send method 完成出站。
